okay, what even is machine learning?

Imagine you want to teach a cat to know if a treat is "yummy" or "yucky". You don't write rules. Instead you show it 100 examples — "this one was yummy, this one was yucky" — and the cat slowly figures out the pattern itself. Next time it sees a new treat, it guesses based on what it learned.

That's machine learning: learning patterns from examples (data) instead of being given explicit rules.

There are two big families you need for finals:

  • Supervised learning — you have labelled examples (treat → yummy/yucky). The model learns to predict the label. Covers: KNN, Naive Bayes, Linear/Logistic Regression, SVM, Decision Trees, Random Forest, Neural Nets.
  • Unsupervised learning — no labels, just data. The model finds hidden structure. Covers: K-Means (grouping), PCA (simplifying).

Two more words your sir loves:

  • Classification = predicting a category (cat / dog / yummy / yucky).
  • Regression = predicting a number (price, grade, temperature).

That's it. Every algorithm below is just a different clever way to do one of these. Let's go. 🌸

1

Naive Bayes

CAT SAYS: You see a small, fluffy, meowing thing. You assume each clue (small, fluffy, meows) is a separate independent hint. Combine all hints using probability → it's a Cat.

The whole idea

For each possible class, ask: "based on these features, how likely is this class?" Multiply probabilities, pick the biggest. It's literally counting and multiplying — no scary math.

Bayes TheoremP(Class | Features) ∝ P(Class) × P(f₁|Class) × P(f₂|Class) × ...

Prior P(Class)

How common is this class overall?
P(Yes)=count(Yes)/total

Likelihood P(f|Class)

How often this feature shows up in this class
P(Sunny|Yes)=count(Sunny&Yes)/count(Yes)

Solved — Play Tennis

Test: Sunny, Cool, High humidity, Strong wind. 14 days: 9 Yes, 5 No.

Priors: P(Yes)=9/14=0.643, P(No)=5/14=0.357

Score(Yes) = 0.643 × (2/9) × (3/9) × (3/9) × (3/9) = 0.00529

Score(No) = 0.357 × (3/5) × (1/5) × (4/5) × (3/5) = 0.02057

No (80%) > Yes (20%) → DO NOT PLAY
⚠ If any probability = 0, the whole product = 0. Fix with Laplace smoothing: add 1 to every count.
Why is it called "Naive"?
It naively assumes all features are independent of each other given the class. Almost never true in reality, but it works astonishingly well and is super fast — perfect for spam filtering and text classification.
Quick Check
What does P(Cool|No) mean?
It's a likelihood: out of the 5 "No" days, the fraction that were Cool = 1/5.

K-Nearest Neighbors (KNN)

CAT SAYS: New song, unknown genre? Find the 3 most similar songs you already know and copy their genre. KNN never 'studies' — it just memorizes everything and compares at the last second.

The whole idea

Lazy learner: no training. Store all data. For a new point, find the K closest points and let them vote.

Euclidean Distanced = √[(x₁−x₂)² + (y₁−y₂)²]
KEffectRisk
1Follows every point exactlyOverfitting (noisy)
LargeVery smooth boundaryUnderfitting
OddAvoids tiesAlways use odd for 2 classes

Solved — classify X=(4,2), K=3

A(1,2)=Cat, B(3,4)=Cat, C(5,1)=Dog, D(4,3)=Dog

d(X,A)=3.00 d(X,B)=2.24 d(X,C)=1.41 d(X,D)=1.00

Nearest 3: D(Dog), C(Dog), B(Cat)

Vote: Dog 2, Cat 1 → X = DOG 🐕
⚠ Curse of dimensionality: with too many features, all distances look equal and KNN breaks. Reduce dimensions first (PCA).
How does K affect classification?
Small K = sensitive to local noise/outliers (can flip the answer). Large K = smoother but may absorb other classes and underfit. The same point can get different labels for K=3 vs K=5 — so K matters and should be tuned with cross-validation.

Linear Regression

CAT SAYS: Track treats eaten vs energy level. Draw the single straight line that best fits all your past dots. That line predicts future energy. 'Best' = smallest total error.

The whole idea

Fit a straight line through the data to predict a number.

Hypothesisŷ = θ₀ + θ₁x₁ + θ₂x₂ + ...
Cost — Mean Squared ErrorJ = (1/n) Σ(yᵢ − ŷᵢ)²

The Normal Equation (closed-form solve)

Instead of gradient descent, you can solve directly:

Normal Equationθ = (XᵀX)⁻¹ Xᵀy

X = design matrix (add a column of 1s for the bias θ₀). This is the exact method your sir's car-price question uses — fully solved in the Exam Vault below.

Why add a column of 1s to the design matrix?
The column of 1s lets the model learn the intercept/bias θ₀ (the base value when all features are 0). Without it, the line is forced through the origin.

Gradient Descent

CAT SAYS: You're blindfolded on a hill and want the lowest valley. Feel the slope under your paws, step downhill a little, repeat. Eventually you reach the bottom = minimum cost.

The whole idea

An iterative way to find the weights that minimize the cost. Each step moves opposite to the slope (gradient).

Update Rulew := w − α × (∂J/∂w)

α = learning rate (step size).

αWhat happens
Too smallCrawls — takes forever to converge
Too largeOvershoots — bounces around, diverges
Just rightSmooth steady descent

Solved — J(w)=w², w₀=4, α=0.1

Gradient of w² is 2w.

1
Start cost = 4² = 16
2
Iter 1: grad=2(4)=8 → w=4−0.1(8)=3.2, cost=3.2²=10.24
3
Iter 2: grad=2(3.2)=6.4 → w=3.2−0.1(6.4)=2.56, cost=2.56²=6.55
Cost: 16 → 10.24 → 6.55 = DECREASING ✓ Gradient descent is working — it's walking toward the minimum.
⚠ In an exam, swap in YOUR given cost function and values. The recipe is always: (1) find gradient, (2) plug into update rule, (3) repeat, (4) check cost is dropping.

Logistic Regression

CAT SAYS: Linear regression gives any number. But 'will the cat knock the glass off? yes/no' needs a probability between 0 and 1. Sigmoid squishes the line into that range.
⚠ Despite the name, this is CLASSIFICATION, not regression!
Sigmoidσ(z) = 1 / (1 + e^(−z)), where z = θ₀ + θ₁x₁ + ...
Cost — Log Loss / Cross-EntropyJ = −(1/n) Σ[ y·log(ŷ) + (1−y)·log(1−ŷ) ]

Decision boundary

σ(z) ≥ 0.5 → Class 1 (when z ≥ 0). σ(z) < 0.5 → Class 0. At z=0, σ=0.5 exactly.

Why Log Loss and not MSE here?
MSE + sigmoid creates a non-convex cost with many local minima where gradient descent gets stuck. Log loss is convex → one global minimum, always reachable.
Quick Check
σ(0) = ?
σ(0)=1/(1+e⁰)=1/2=0.5 — the decision boundary.

Regularization (L1 & L2)

CAT SAYS: Ridge: 'keep all your toys but play gentler with each.' Lasso: 'pick your favourite toys, throw the rest out.' Both stop the model obsessing over training data.
Ridge (L2)J = Σ(y−ŷ)² + λ Σ wⱼ²
Lasso (L1)J = Σ(y−ŷ)² + λ Σ |wⱼ|
Ridge (L2)Lasso (L1)
WeightsShrink toward 0, never reach 0Can become exactly 0
Feature selectionNoYes (drops features)
Use whenAll features matterMany useless features

The λ trap (your sir asked this!)

λ very HIGH → penalty too strong → weights crushed to ~0 → model too simple → UNDERFITTING (high bias).

λ very LOW (→0) → almost no penalty → weights grow large → model too complex → OVERFITTING (high variance).

Goal: pick a balanced λ via cross-validation — the sweet spot between under and over.
500 features, only ~20 useful. Ridge or Lasso?
Lasso (L1) — it zeros out the 480 useless features automatically (feature selection), giving a clean sparse model. Ridge would keep all 500.

Support Vector Machine (SVM)

CAT SAYS: Separate cats from dogs with a line. Don't draw just any line — draw the one with the WIDEST gap on both sides. The animals sitting right on the gap edges are the 'support vectors'.
TermMeaning
HyperplaneThe boundary: w·x + b = 0
Support VectorsClosest points — they alone define the boundary
MarginGap width = 2/|w|
Hard marginPerfect separation, no errors allowed
Soft marginAllows some errors (slack ξ, penalty C)
ObjectiveMinimize ½|w|² → maximizes the margin 2/|w|

Kernel trick

CAT SAYS: Data tangled in 2D? Lift it into 3D with a magic function — now a flat sheet separates it. RBF kernel is your default go-to.
KernelUse for
LinearAlready separable by a line
PolynomialCurved boundaries
RBF ⭐Most cases — the default
Delete a non-support-vector point — what happens?
Nothing. Only support vectors define the hyperplane. Removing any other point leaves the model identical. That's why SVM is memory-efficient.

Decision Tree

CAT SAYS: Raining? → umbrella. Cold? → jacket. A flowchart of yes/no questions ending in an answer. Each question is chosen to reduce confusion (impurity) the most.
Entropy (disorder)H(S) = −Σ pᵢ log₂(pᵢ)
Information GainIG = H(S) − Σ (|Sᵥ|/|S|) H(Sᵥ)
Gini Impurity (faster)Gini = 1 − Σ pᵢ²

H=0 → pure (all same class). H=1 → max confusion. Pick the feature with the highest Information Gain to split on.

Solved — Outlook split

14 days (9Y,5N). Sunny(2Y,3N), Overcast(4Y,0N), Rain(3Y,2N).

1
H(S)=−(9/14)log₂(9/14)−(5/14)log₂(5/14)=0.940
2
H(Sunny)=0.971, H(Overcast)=0 (pure!), H(Rain)=0.971
3
Weighted=(5/14)0.971+(4/14)0+(5/14)0.971=0.694
4
IG=0.940−0.694=0.246
Do this for every feature → highest IG becomes the root.
Entropy vs Gini?
Both measure impurity. Entropy uses logs (slightly more sensitive). Gini uses squares (faster, no log). Results are nearly identical — sklearn defaults to Gini. Either is fine in exams.

Random Forest & Ensembles

CAT SAYS: One cat deciding if it's safe outside = risky. Ask 100 cats and take the majority vote = reliable. Random Forest = 100 decision trees, each on different random data and features, voting together.
MethodHowReduces
BaggingParallel trees on random samples, voteVariance
BoostingSequential, each fixes prior's errorsBias

Random Forest = Bagging + random feature subset at each split (m ≈ √features). Diverse trees → averaging cancels their individual mistakes.

OOB Score

~1/3 of samples skipped per tree → use them as free validation.

Feature Importance

How much each feature reduces impurity across all trees = its importance ranking.
Why beat a single tree?
A single deep tree overfits (high variance). RF averages many diverse trees, slashing variance while keeping bias low. Plus free OOB validation and feature importance. One of the best off-the-shelf models.

K-Means Clustering

CAT SAYS: Scatter cats in a room, drop K food bowls randomly. Each cat goes to the nearest bowl. Move each bowl to the center of its cats. Repeat until cats stop switching. Done.
Unsupervised — no labels. It groups similar points by distance.
1
Pick K, place K centroids randomly
2
Assign: each point → nearest centroid
3
Update: move centroid to mean of its points
4
Repeat 2–3 until nothing moves
Objective (minimize)WCSS = Σ ||xᵢ − μₖ||²

Choosing K — Elbow method

Plot K vs WCSS. It drops then flattens. The "elbow" bend = best K.

⚠ Sensitive to initialization (use K-means++), assumes round equal clusters, hurt by outliers.
Quick Check
K-Means is which type of learning?
Unsupervised — no labels, it finds hidden groups purely from distances.

Principal Component Analysis (PCA)

CAT SAYS: 50 features describe a cat, most overlapping. PCA finds the 2–3 directions where cats differ the MOST and keeps only those. 50 numbers → 3 numbers, 95% of the info kept.
1
Center data (subtract mean)
2
Covariance matrix: Cov = (1/n−1) ZᵀZ
3
Find eigenvalues + eigenvectors
4
Biggest eigenvalue = PC1 (most variance)
5
Project onto top components
Variance explainedby PCk = λₖ / Σλ (keep until ~95%)

Your sir's exact PCA problem (size/rooms) is fully solved in the Exam Vault.

Eigenvalue = 0 means?
Zero variance in that direction — that component carries no information, discard it. The data effectively lives in fewer dimensions.

Neural Networks

CAT SAYS: Brain cells in layers. Signal goes in, each neuron transforms and passes it forward (forward pass). Wrong answer? Send the error backward to blame and fix each weight (backprop).
ActivationUse
SigmoidOutput (binary)
ReLU = max(0,z)Hidden layers (default)
SoftmaxOutput (multi-class)
Forward passzˡ = Wˡaˡ⁻¹ + bˡ ; aˡ = f(zˡ)

Solved forward pass

x=[0.5,0.8], hidden weights given, sigmoid activation.

z1=0.57 → σ=0.639 ; z1=0.83 → σ=0.696

z2 = 0.7(0.639)+0.8(0.696)+0.1 = 1.104 → σ(1.104)=0.751

Output 0.751 > 0.5 → Class 1
Backprop = chain rule∂L/∂W = ∂L/∂a · ∂a/∂z · ∂z/∂W ; σ'(z)=σ(z)(1−σ(z))
Why need activation functions?
Without them, stacking layers is just one big linear function — can't learn curves like XOR. Non-linear activations let the network learn complex patterns. ReLU avoids vanishing gradients, so it's preferred in hidden layers.

Model Evaluation

CAT SAYS: Your disease detector says 'healthy' to everyone and scores 99% accuracy — but missed every sick cat. Accuracy lies on imbalanced data. Look at the full confusion matrix.
Pred PositivePred Negative
Actual PosTP ✓FN (missed)
Actual NegFP (false alarm)TN ✓
MetricFormulaReads as
Accuracy(TP+TN)/alloverall correct
PrecisionTP/(TP+FP)of predicted-yes, how many right
Recall / TPRTP/(TP+FN)of actual-yes, how many caught
F12PR/(P+R)balance of P & R

The 0/1 loss (your sir asked!)

0/1 loss = 0 if prediction correct, 1 if wrong.

L(y, ŷ) = 1 if y ≠ ŷ, else 0

Purpose: directly counts misclassifications — the most intuitive error measure (it IS the error rate). Problem: it's non-differentiable and non-convex, so you can't run gradient descent on it. Used for evaluation, not training.

96% accuracy on cancer detection — why not celebrate?
If ~96% of patients are healthy, a model that says "healthy" to everyone scores 96% while catching ZERO cancers. Accuracy is misleading on imbalanced data. Fix: report Precision/Recall/F1/AUC, prioritise Recall (don't miss cancers), use the confusion matrix, and resample/oversample the minority class (e.g. SMOTE).
Quick Check
Detector catches 90/100 cancers, falsely flags 50 healthy. Recall = ?
Recall=TP/(TP+FN)=90/(90+10)=90%. The 50 false alarms hurt Precision, not Recall.

V6 · Regularization λ + 0/1 loss + accuracy trap

The three short Q1 theory parts your sir asked.

▸ Tap to reveal full solution

I. Impact of λ

Very high λ: over-penalizes weights → forces them near 0 → model too simple → underfitting (high bias). Very low λ: barely penalizes → weights grow → model too complex → overfitting (high variance). Balanced λ via cross-validation = best.

II. 0/1 loss

L=0 if correct, 1 if wrong. Formula: L(y,ŷ)=1 if y≠ŷ else 0. Purpose: directly counts misclassifications = intuitive error rate. But non-differentiable/non-convex → used for evaluation, not gradient-based training.

III. 96% cancer accuracy

On imbalanced data (mostly healthy), predicting "healthy" always scores ~96% yet catches no cancer. Accuracy misleads. Use Precision/Recall/F1/AUC, prioritise Recall, check the confusion matrix, and oversample the minority class (SMOTE).
