hi. sit down, breathe. let's actually get this.

Machine learning sounds scary but here's the honest version: you show a computer lots of examples, and it figures out the pattern by itself — instead of you writing the rules.

Imagine teaching a cat which treats are yummy. You don't explain chemistry. You just show it 100 treats labelled yummy/yucky, and it learns to guess. That's it. That's ML.

Two families you'll meet:

🟦 Supervised — you have the answers (labels). The model learns to predict them. (Linear/Logistic Regression, KNN, Naive Bayes, Decision Tree, Random Forest, ANN.)
🟪 Unsupervised — no answers. The model finds hidden structure. (K-Means groups things, PCA simplifies things.)

And two jobs: Classification = predict a category (cat/dog). Regression = predict a number (price/grade).

Every topic below has: a cat analogy, a plain-English teach, a "what do these symbols mean?" decoder 🔤, a worked example, scenario practice, and whiteboard questions you solve first then reveal. Your finals are mostly numerical — so do the whiteboard ones with a pen. Let's go. 🐾

Linear Regression

before mid
🐱 CAT SAYS: You note how many treats you eat vs how happy you feel. You draw ONE straight line through your dots. That line lets you guess your happiness for any number of treats. 'Best line' = the one closest to all dots.

This is your first real algorithm, so slow down with me. We want to predict a number (a price, a grade, happiness). We believe the answer follows a straight line. The whole job is: find the perfect line.

A line needs two things — how steep it is (slope) and where it starts (intercept). In ML we call them w and b. That's literally all the model is: ŷ = wx + b.

To know if our line is good, we measure how far off our guesses are. That's the cost (MSE). Small cost = good line.

The model (a straight line)ŷ = wx + b
Cost — Mean Squared ErrorJ(w,b) = (1/2m) · Σ(ŷᵢ − yᵢ)²
🔤 what do these symbols mean?
ŷ'y-hat' — the value your model PREDICTS
ythe REAL answer from your data
xthe input you're given (e.g. hours studied)
wweight / slope — how steep the line is
bbias / intercept — where the line crosses the y-axis
mhow many data examples you have
Σ'sigma' — just means 'add them all up'
Jthe cost — how WRONG the line is (we want this small)

✏️ Worked Example

1
Data: x=[1,2,3], y=[2,4,6]. Try the line w=2, b=0 (so ŷ=2x).
2
Predict: ŷ = 2(1)=2, 2(2)=4, 2(3)=6
3
Errors: (2−2)=0, (4−4)=0, (6−6)=0 — all zero!
4
Cost: J = (1/2·3)(0²+0²+0²) = 0 → perfect fit, the line hits every point.
If instead w=1: predictions 1,2,3 → errors −1,−2,−3 → J=(1/6)(1+4+9)=2.33. Bigger cost = worse line.

🎯 Scenario Practice

Scenario: A model gets MSE = 0 on training data but MSE = 18 on test data. What's going on?
show how to answer it ▸
This is overfitting. The line memorised the training points perfectly (train MSE=0) but fails on new unseen data (test MSE=18). The model didn't learn the real pattern, it memorised noise. Fix: simpler model, more data, or regularization.

📝 Your Turn — Whiteboard

✍️ Whiteboard #1
Data: x=[1,2,3], y=[1,2,3]. Using the line f(x)=0.5x, calculate the MSE. (use J=(1/2m)Σ(ŷ−y)²)
I tried it — show the solution ▸
Predictions: 0.5, 1, 1.5. Errors: 0.5−1=−0.5, 1−2=−1, 1.5−3=−1.5. Squared: 0.25, 1, 2.25. Sum=3.5. J=(1/2·3)(3.5)=3.5/6 ≈ 0.583.
✍️ Whiteboard #2

I tried it — show the solution ▸
Predictions: 2(0)+1=1, 2(1)+1=3, 2(2)+1=5. Errors: 0,0,0. MSE = 0 — perfect fit! The line passes through every point.

Gradient Descent

before mid
🐱 CAT SAYS: You're blindfolded on a foggy hill, trying to reach the lowest valley. You feel which way is downhill under your paws, take a small step that way, and repeat. Eventually you're at the bottom = lowest cost.

Last topic we guessed w and b. But how does the computer FIND the best ones? Gradient descent. It starts with random w,b and keeps nudging them downhill on the cost.

The gradient is just the slope of the cost — it points uphill, so we step the OPPOSITE way (that's the minus sign). The learning rate α decides how big each step is.

⚠️ The one rule examiners love: update w and b simultaneously — compute both changes using the OLD values, then apply them together.

Update rules (do both together!)w = w − α·(∂J/∂w) b = b − α·(∂J/∂b)
The gradients∂J/∂w = (1/m)Σ(ŷᵢ−yᵢ)xᵢ ∂J/∂b = (1/m)Σ(ŷᵢ−yᵢ)
🔤 what do these symbols mean?
α'alpha' — the learning rate = step size
∂J/∂wthe gradient = slope of cost w.r.t. w (which way is uphill)
w, bthe line's slope and intercept being improved
ŷ−ythe error (prediction minus truth)
mnumber of examples
:='becomes' — assign the new value

✏️ Worked Example

1
Data: x=[1,2,3], y=[2,4,6]. Start w=0, b=0, α=0.1.
2
Predict: all ŷ=0. Errors: 0−2=−2, 0−4=−4, 0−6=−6.
3
Gradient w: (1/3)[(−2)(1)+(−4)(2)+(−6)(3)] = (1/3)(−28) = −9.33
4
Gradient b: (1/3)(−2−4−6) = −4
5
Update: w = 0 − 0.1(−9.33) = 0.933, b = 0 − 0.1(−4) = 0.4
After 1 step w went 0→0.933, b went 0→0.4. Repeat many times and they settle at w=2, b=0 (the perfect line).

🎯 Scenario Practice

Scenario: During training the cost keeps jumping up and down and never settles. What's wrong and how do you fix it?
show how to answer it ▸
The learning rate α is too high — each step overshoots the minimum and bounces to the other side. 'Train longer' won't help. Fix: lower α (e.g. 0.1 → 0.01) so steps are smaller and descent is smooth.

📝 Your Turn — Whiteboard

✍️ Whiteboard #1
Start at w=2, ∂J/∂w=1.5, α=0.1. Find the new w after one update.
I tried it — show the solution ▸
w = w − α·(∂J/∂w) = 2 − 0.1(1.5) = 2 − 0.15 = 1.85.
✍️ Whiteboard #2
w=3, b=1, α=0.05, ∂J/∂w=2, ∂J/∂b=−1. Find new w and b (simultaneous update).
I tried it — show the solution ▸
w = 3 − 0.05(2) = 3 − 0.1 = 2.9. b = 1 − 0.05(−1) = 1 + 0.05 = 1.05. Both computed from the original values, then applied together.

K-Nearest Neighbors

before mid
🐱 CAT SAYS: New song, unknown genre? You find the 3 songs most similar to it that you already know, and copy whatever genre most of them are. KNN never 'studies' — it just remembers everything and compares at the last second.

This is the friendliest algorithm. To label a new point, look at its K closest neighbours and take a vote. That's it.

'Closest' is measured by distance (usually Euclidean — straight-line distance). Smaller distance = more similar.

Two rules examiners test: (1) always use an odd K for 2 classes so you can't get a tie, and (2) always normalize features first, or a big-numbered feature (like salary) will drown out a small one (like age).

Euclidean distanceD = √[(x₁−x₂)² + (y₁−y₂)²]
Min-Max normalizationx_scaled = (x − x_min)/(x_max − x_min)
🔤 what do these symbols mean?
Ddistance between two points (smaller = more similar)
Khow many neighbours get to vote
(x₁,y₁)coordinates of point 1 (the new point)
(x₂,y₂)coordinates of point 2 (a training point)
x_min, x_maxsmallest & largest value of that feature
lazy learnerKNN does no training — it just stores data

✏️ Worked Example

1
Classify P=(3,2) among A(1,2)=C1, B(2,3)=C1, C(3,3)=C2, D(6,5)=C2, E(7,7)=C2.
2
Distances: d(A)=2.00, d(B)=1.41, d(C)=1.00, d(D)=4.24, d(E)=6.40
3
Sort: C(1.00,C2) · B(1.41,C1) · A(2.00,C1) · D(4.24,C2) · E(6.40,C2)
4
K=3: C,B,A → C2=1, C1=2 → Class 1
5
K=5: all → C1=2, C2=3 → Class 2
Notice K flipped the answer! Small K listens to the nearest few; large K listens to a wider crowd. That's why K matters.

🎯 Scenario Practice

Scenario: A KNN model uses Salary (range 30,000–90,000) and Age (range 20–60) without normalizing. Why will it give bad predictions?
show how to answer it ▸
Distance is dominated by Salary because its numbers are thousands of times larger — Age barely affects the result, so neighbours are chosen almost entirely on salary. Fix: normalize both to [0,1] with Min-Max so they contribute equally.

📝 Your Turn — Whiteboard

✍️ Whiteboard #1
Points A(1,1)=Class0, B(2,2)=Class0, C(4,4)=Class1, D(5,5)=Class1. Classify P(3,3) with K=3 using Euclidean distance.
I tried it — show the solution ▸
d(A)=√8=2.83, d(B)=√2=1.41, d(C)=√2=1.41, d(D)=√8=2.83. Three nearest: B(C0), C(C1), and A(C0) → votes C0=2, C1=1 → Class 0.
✍️ Whiteboard #2
Feature Age=[20,40,60]. Normalize with Min-Max. What's the scaled value of Age=40?
I tried it — show the solution ▸
(40−20)/(60−20) = 20/40 = 0.5. (And 20→0.0, 60→1.0.)

Regularization — L1 & L2

before mid
🐱 CAT SAYS: Ridge says 'keep all your toys but play gentler with each.' Lasso says 'pick your favourites and throw the useless ones OUT.' Both stop the model from obsessing over training data.

When a model overfits, its weights often blow up huge. Regularization adds a penalty for big weights to the cost, forcing the model to stay humble and simple.

Ridge (L2) squares the weights in the penalty → shrinks them small but never to zero (keeps all features). Lasso (L1) uses absolute value → can push weights to exactly zero, deleting useless features automatically.

Ridge (L2)J = Σ(y−ŷ)² + λ·Σ wⱼ²
Lasso (L1)J = Σ(y−ŷ)² + λ·Σ |wⱼ|
🔤 what do these symbols mean?
λ'lambda' — regularization strength (how hard we punish big weights)
wⱼthe j-th weight/coefficient
Σwⱼ²sum of squared weights (L2 penalty)
Σ|wⱼ|sum of absolute weights (L1 penalty)
λ=0no penalty → plain regression
λ hugeweights crushed → too simple → underfit

✏️ Worked Example

The λ trap (examiners love this): λ too HIGH → weights → 0 → model too simple → underfitting. λ too LOW → weights grow → too complex → overfitting. Pick the balanced λ with cross-validation.

Memory hook: L1 = Lasso = Lops off features (can be zero). L2 = Ridge = Reduces (never zero).

🎯 Scenario Practice

Scenario: You have 100 features but suspect only ~10 actually matter. Ridge or Lasso, and why?
show how to answer it ▸
Lasso (L1). It performs automatic feature selection — driving the ~90 useless weights to exactly zero, leaving a clean sparse model with only the important features. Ridge would keep all 100 (just smaller).

📝 Your Turn — Whiteboard

✍️ Whiteboard #1
In one sentence: what is the key difference between Ridge and Lasso?
I tried it — show the solution ▸
Ridge (L2) shrinks weights toward zero but never exactly zero (keeps all features), while Lasso (L1) can shrink weights to exactly zero (removes features = automatic feature selection).
✍️ Whiteboard #2
A model has train accuracy 99% and test accuracy 60%. Name the problem and two fixes involving λ.
I tried it — show the solution ▸
Problem: overfitting. Fixes: add regularization / increase λ to shrink weights and simplify the model; also more data or fewer features. (But don't set λ too high or it underfits.)

Logistic Regression

finals
🐱 CAT SAYS: Linear regression gives any number. But 'will the cat knock the glass off — yes or no?' needs a probability between 0 and 1. The sigmoid function squishes any number into that range.

Don't let the name fool you — this does classification (yes/no), not regression. We compute a normal weighted sum z = w·x + b, then squash it through the sigmoid so the output is a probability between 0 and 1.

If the probability ≥ 0.5 → class 1, else class 0. The point where z=0 (probability exactly 0.5) is the decision boundary.

Sigmoid (squash into 0–1)σ(z) = 1 / (1 + e^(−z))
The score before squashingz = w₀ + w₁x₁ + w₂x₂ + …
Cost — Log LossJ = −(1/m)Σ[ y·log(ŷ) + (1−y)·log(1−ŷ) ]
🔤 what do these symbols mean?
σ(z)'sigma of z' — the sigmoid, outputs a probability 0–1
zthe raw weighted sum (any number)
eEuler's number ≈ 2.718 (just a constant)
ŷpredicted probability = σ(z)
w₀the bias term (same as b)
≥ 0.5threshold: above → class 1, below → class 0

✏️ Worked Example

1
Trained model: w₀=−4, w₁=0.05 (x = hours studied). Predict pass/fail.
2
x=60: z = −4 + 0.05(60) = −1 → σ(−1)=0.269 → FAIL (below 0.5)
3
x=80: z = −4 + 0.05(80) = 0 → σ(0)=0.5 → exactly on the boundary
4
x=100: z = −4 + 0.05(100) = 1 → σ(1)=0.731 → PASS
Key fact to memorise: σ(0)=0.5 always. That's the decision line.

🎯 Scenario Practice

Scenario: Why do we use Log Loss instead of MSE to train logistic regression?
show how to answer it ▸
With sigmoid, MSE creates a non-convex cost (lots of bumps/local minima) where gradient descent gets stuck. Log Loss is convex — one single global minimum — so gradient descent reliably finds the best weights.

📝 Your Turn — Whiteboard

✍️ Whiteboard #1
Model: z = −3 + 1·x. For x=2, compute the probability and classify (threshold 0.5). (σ(−1)≈0.27)
I tried it — show the solution ▸
z = −3 + 1(2) = −1. σ(−1) ≈ 0.27. Since 0.27 < 0.5 → Class 0.
✍️ Whiteboard #2
What is σ(0), and why does it matter?
I tried it — show the solution ▸
σ(0) = 1/(1+e⁰) = 1/(1+1) = 0.5. It's the decision boundary — exactly between the two classes.

Naive Bayes

finals
🐱 CAT SAYS: You see a small, fluffy, meowing thing. You treat each clue (small, fluffy, meows) as a separate independent hint, multiply their probabilities, and conclude: Cat. That independence assumption is the 'naive' part.

This is pure counting and multiplying — no calculus, promise. For each possible class you compute: how common is this class (prior) × how typical are these features in that class (likelihoods). Biggest result wins.

'Naive' = we pretend all features are independent. Unrealistic, but it works great and is lightning fast (spam filters use it).

Bayes (proportional form)P(Class | X) ∝ P(Class) · P(f₁|Class) · P(f₂|Class) · …
🔤 what do these symbols mean?
P(Class)the prior — how often this class appears overall
P(f|Class)likelihood — how often this feature shows up within that class
∝'proportional to' — we skip the shared denominator and just compare
Xthe new thing's features (e.g. Sunny, Cool, High)
Πmultiply them all together

✏️ Worked Example

Solved A — Play Tennis

1
Priors: P(Yes)=9/14, P(No)=5/14.
2
Score(Yes)=0.643·(2/9)·(3/9)·(3/9)·(3/9)=0.00529
3
Score(No)=0.357·(3/5)·(1/5)·(4/5)·(3/5)=0.02057
4
No (80%) > Yes (20%) → DON'T PLAY

Solved B — your slide's CAR example

10 cars (5 Stolen=Yes, 5 No). Predict for {Red, Imported, Sports}.

1
Among Yes: P(Red)=3/5, P(Imported)=3/5, P(Sports)=4/5
2
Among No: P(Red)=2/5, P(Imported)=2/5, P(Sports)=2/5
3
Score(Yes)=0.5·0.6·0.6·0.8 = 0.144
4
Score(No)=0.5·0.4·0.4·0.4 = 0.032
5
0.144 > 0.032 → STOLEN = YES
⚠️ Zero trap: in the FRUIT slide example, P(Long|Mango)=0/650=0, which kills Mango's whole score (×0=0). Banana wins (0.219). Fix zeros with Laplace smoothing (+1 to every count).

🎯 Scenario Practice

Scenario: A spam filter sees 'free', 'winner', 'click'. Explain how Naive Bayes decides spam vs not-spam.
show how to answer it ▸
It computes Score(spam) = P(spam)·P(free|spam)·P(winner|spam)·P(click|spam), and the same for not-spam, then picks the bigger. It's 'naive' because it assumes the three words appear independently of each other given the class.

📝 Your Turn — Whiteboard

✍️ Whiteboard #1
Tennis data. New day: Outlook=Overcast, Temp=Hot, Humidity=High, Wind=Weak. Given P(Overcast|Yes)=4/9, P(Hot|Yes)=2/9, P(High|Yes)=3/9, P(Weak|Yes)=6/9 and P(Overcast|No)=0/5. Which class wins, and why instantly?
I tried it — show the solution ▸
P(Overcast|No)=0/5=0, so Score(No)=0 immediately. Score(Yes) is positive. → PLAY (Yes). (Overcast never appeared on a No day, so the No score zeroes out — this is exactly why Laplace smoothing exists.)
✍️ Whiteboard #2
Why is the algorithm called 'naive'?
I tried it — show the solution ▸
Because it naively assumes all features are independent of each other given the class — almost never true in real life, but it still works remarkably well and is very fast.

Decision Tree (Entropy)

finals
🐱 CAT SAYS: Raining? → take umbrella. Cold? → wear jacket. A decision tree is a flowchart of yes/no questions ending in an answer. Each question is chosen to clear up the most confusion.

We build a tree of questions. At each step we ask: 'which feature, if I split on it, makes the groups PUREST (least mixed)?' We measure mixed-ness with entropy, and the best split is the one with the highest information gain.

Entropy = 0 means a group is pure (all one class). Entropy = 1 means maximally mixed (50/50). Lower entropy after the split = better question.

Entropy (mixed-ness)H(S) = −Σ pᵢ·log₂(pᵢ)
Information GainIG = H(S) − Σ (|Sᵥ|/|S|)·H(Sᵥ)
🔤 what do these symbols mean?
H(S)entropy of set S (0=pure, 1=most mixed)
pᵢfraction of class i in the group (e.g. 9/14)
log₂log base 2 (because info is measured in bits)
IGinformation gain — how much entropy the split removed
Sᵥa child group after splitting on a feature value
|Sᵥ|/|S|that child's share of the data (weight)

✏️ Worked Example

1
14 days: 9 Yes, 5 No. H(S) = −(9/14)log₂(9/14) − (5/14)log₂(5/14) = 0.940
2
Split on Outlook → Sunny(2Y,3N), Overcast(4Y,0N), Rain(3Y,2N).
3
H(Sunny)=0.971, H(Overcast)=0 (pure!), H(Rain)=0.971
4
Weighted = (5/14)0.971 + (4/14)0 + (5/14)0.971 = 0.694
5
IG(Outlook) = 0.940 − 0.694 = 0.246
Compute IG for every feature → the one with the HIGHEST IG becomes the question at that node.

🎯 Scenario Practice

Scenario: After a split, all child nodes are pure (each child is 100% one class). Is that a good or bad split? What's the information gain like?
show how to answer it ▸
Excellent split. Pure children have entropy = 0, so the information gain is maximised (parent entropy − 0). The tree-builder will strongly prefer this split.

📝 Your Turn — Whiteboard

✍️ Whiteboard #1
A group has 6 Yes and 4 No (10 total). Calculate its entropy. (log₂(0.6)≈−0.737, log₂(0.4)≈−1.322)
I tried it — show the solution ▸
H = −(6/10)log₂(6/10) − (4/10)log₂(4/10) = −(0.6)(−0.737) − (0.4)(−1.322) = 0.442 + 0.529 = 0.971 (quite mixed).
✍️ Whiteboard #2
A group is 100% Yes (8 Yes, 0 No). What is its entropy and why?
I tried it — show the solution ▸
H = −(1)log₂(1) − 0 = −(1)(0) = 0. A pure group has zero entropy — no confusion at all.

Random Forest

finals
🐱 CAT SAYS: One cat deciding if it's safe outside = risky. Ask 100 cats and take the majority vote = reliable. Random Forest = many decision trees, each trained on different random data & features, all voting together.

(Your syllabus says: concept only, no numerical — so just understand the why.)

A single decision tree easily overfits. Random Forest builds many trees, each on a random sample of the data (bagging) and a random subset of features at each split. Then they vote. Their individual mistakes cancel out, so the forest is far more stable.

Features tried per splitm ≈ √(total features)
🔤 what do these symbols mean?
BaggingBootstrap AGGregating — train trees on random samples (with replacement)
OOBOut-Of-Bag — the ~1/3 of samples a tree didn't see, used as free validation
varianceerror from being too sensitive to data — RF reduces this by averaging
mnumber of features randomly considered at each split

✏️ Worked Example

Why it beats one tree: one deep tree has low bias but HIGH variance (overfits). Averaging many diverse trees keeps the low bias but slashes the variance → better on new data. Bonus: free OOB validation + feature importance.

🎯 Scenario Practice

Scenario: Your single decision tree gets 100% train accuracy but poor test accuracy. How does switching to a Random Forest help?
show how to answer it ▸
The single tree overfit (memorised the training data = high variance). A Random Forest averages many different trees, each seeing different data/features, so their overfitting cancels out — variance drops and test accuracy improves, while bias stays low.

📝 Your Turn — Whiteboard

✍️ Whiteboard #1
In your own words: what two kinds of 'randomness' make a Random Forest's trees different from each other?
I tried it — show the solution ▸
(1) Random data — each tree trains on a different bootstrap sample (random rows, with replacement). (2) Random features — at each split a tree only considers a random subset of features (≈√total). Together these make the trees diverse so their errors cancel.

K-Means Clustering

finals
🐱 CAT SAYS: Scatter cats in a room, drop K food bowls randomly. Each cat walks to its nearest bowl. Move each bowl to the centre of its cats. Repeat until no cat switches bowls. Done — you've found the groups.

This is unsupervised — there are NO labels. We just want to group similar points into K clusters. We repeat two moves until stable: (1) assign each point to the nearest centre, (2) move each centre to the average of its points.

Distance (assign step)D = √[(x₁−x₂)² + (y₁−y₂)²]
Objective (minimize)WCSS = Σ ||xᵢ − μₖ||²
🔤 what do these symbols mean?
Khow many clusters you want (you choose this)
centroid μₖthe centre point of cluster k
WCSSWithin-Cluster Sum of Squares — total spread inside clusters (smaller=tighter)
||xᵢ − μₖ||distance from a point to its cluster centre
convergencewhen centroids stop moving = done

✏️ Worked Example

1
6 points, K=2. Start centroids C1=(1,1), C2=(5,7).
2
Assign by nearest centroid → Cluster1={A,B,C}, Cluster2={D,E,F}.
3
Update C1 = mean of {A,B,C} = (1.83, 2.33)
4
Update C2 = mean of {D,E,F} = (4.33, 5.67)
5
Repeat assign→update until nothing moves.
Choosing K — Elbow method: plot K vs WCSS; it drops then flattens. The bend (elbow) is the best K.

🎯 Scenario Practice

Scenario: You run K-Means twice with the same K and get two different clusterings. Why, and how do you reduce this?
show how to answer it ▸
K-Means is sensitive to its random initial centroids — different starts can land in different local minima. Fix: use K-means++ initialization and/or run it several times and keep the result with the lowest WCSS.

📝 Your Turn — Whiteboard

✍️ Whiteboard #1
Is K-Means supervised or unsupervised? Explain in one line.
I tried it — show the solution ▸
Unsupervised — there are no labels; it groups points purely by distance/similarity.
✍️ Whiteboard #2
Centroid C1 owns points (2,2), (4,4), (3,0). Where does C1 move in the update step?
I tried it — show the solution ▸
New C1 = mean = ((2+4+3)/3, (2+4+0)/3) = (9/3, 6/3) = (3, 2).

Principal Component Analysis

finals
🐱 CAT SAYS: 50 numbers describe a cat, most overlapping. PCA finds the 2–3 directions where cats differ the MOST and keeps only those. You go from 50 numbers to 3, keeping ~95% of the info. (Prof said FOCUS here!)

PCA shrinks many features into a few, keeping the directions of biggest variance (spread). The recipe is always the same six steps. Eigenvalues tell you how much variance each direction holds — big eigenvalue = important, zero eigenvalue = useless (drop it).

Covariance matrixCov = (1/(n−1))·ZZᵀ
Eigenvalue equationdet(Cov − λI) = 0 → then (Cov − λI)v = 0
Variance explainedby PCₖ = λₖ / Σλ
🔤 what do these symbols mean?
Zthe mean-centred data (each value minus its column mean)
ZᵀZ transposed (rows↔columns)
Covcovariance matrix — how features vary together
λ'lambda' = eigenvalue = variance along a component
veigenvector = the direction of that component
detdeterminant (for the 2×2 it's ad−bc)
Iidentity matrix [[1,0],[0,1]]
nnumber of data points

✏️ Worked Example

Solved (prof's focus example)

1
Data: X=[1,3,5], Y=[2,6,10]. Means: X̄=3, Ȳ=6.
2
Centre: Xc=[−2,0,2], Yc=[−4,0,4].
3
Covariance = (1/2)ZZᵀ = [[4, 8],[8, 16]].
4
Eigenvalues: (4−λ)(16−λ)−64=0 → λ²−20λ=0 → λ(λ−20)=0 → λ₁=20, λ₂=0.
5
Eigenvector for λ=20: (4−20)v₁+8v₂=0 → v₂=2v₁ → [1,2] normalized = [0.447, 0.894].
6
Result: PC1 holds 20/(20+0)=100% of variance. Drop PC2 (λ=0).
⚠️ Always mean-centre first (step 2). Skip it and PC1 wrongly points at the mean instead of the spread.

🎯 Scenario Practice

show how to answer it ▸
Total = 100. PC1 = 60%. PC1+PC2 = (60+25)/100 = 85%. So keep 2 components.

show how to answer it ▸
Zero variance in that direction — the component carries no information. Drop it. The data really lives in fewer dimensions.

Neural Network — Forward Pass

finals
🐱 CAT SAYS: Brain cells stacked in layers. A signal comes in, each neuron multiplies it by weights, adds a bias, squashes it, and passes it forward. Forward propagation = the signal travelling front-to-back to make a prediction.

(Syllabus: forward propagation only — so we focus on getting the signal to the output.)

Each neuron does two things: a weighted sum (z = Wa + b), then an activation (squash z, e.g. sigmoid). Stack layers and the output of one becomes the input of the next. That's a forward pass.

Weighted sumzˡ = Wˡ·aˡ⁻¹ + bˡ
Activation (sigmoid)aˡ = σ(zˡ) = 1/(1+e^(−zˡ))
🔤 what do these symbols mean?
zˡthe weighted sum at layer l (before squashing)
aˡthe activation/output of layer l (after squashing)
aˡ⁻¹output of the PREVIOUS layer (the input to this one)
Wˡweight matrix for layer l
bˡbias for layer l
σsigmoid activation (squashes to 0–1)

✏️ Worked Example

1
Input x=[0.5, 0.8]. Hidden weights W1=[[0.3,0.4],[0.5,0.6]], b1=[0.1,0.1].
2
Hidden z: 0.3(0.5)+0.4(0.8)+0.1=0.57 ; 0.5(0.5)+0.6(0.8)+0.1=0.83
3
Hidden a: σ(0.57)=0.639 ; σ(0.83)=0.696
4
Output: z = 0.7(0.639)+0.8(0.696)+0.1 = 1.104 → σ(1.104)=0.751
Output 0.751 > 0.5 → predict Class 1. That whole left-to-right computation is one forward pass.

🎯 Scenario Practice

Scenario: Why does a neural network need activation functions like sigmoid or ReLU? What if we removed them?
show how to answer it ▸
Without activations, every layer is just a linear transformation, and stacking linear on linear is still one big linear function — it could never learn curves or patterns like XOR. Activations add non-linearity, which is what lets deep networks learn complex relationships.

Evaluation Metrics

finals
🐱 CAT SAYS: A disease detector that says 'healthy' to everyone scores 99% accuracy — but misses every sick cat. Accuracy lies on imbalanced data. Always read the full confusion matrix.

After classifying, we sort predictions into four boxes (the confusion matrix) and compute metrics. The trap: on imbalanced data, accuracy can look amazing while the model is useless.

Accuracy(TP + TN) / (TP + TN + FP + FN)
PrecisionTP / (TP + FP)
Recall (TPR)TP / (TP + FN)
F1 score2·(P·R)/(P + R)
🔤 what do these symbols mean?
TPTrue Positive — said YES, was YES ✓
TNTrue Negative — said NO, was NO ✓
FPFalse Positive — said YES, was NO (false alarm)
FNFalse Negative — said NO, was YES (missed it!)
Precisionof everything I flagged YES, how much was right
Recallof all real YES cases, how many I caught

✏️ Worked Example

1
Given: TP=60, TN=30, FP=10, FN=0. Total=100.
2
Accuracy = (60+30)/100 = 90%
3
Precision = 60/(60+10) = 85.7%
4
Recall = 60/(60+0) = 100% (caught every positive!)
5
F1 = 2(0.857·1.0)/(0.857+1.0) = 92.3%

🎯 Scenario Practice

Scenario: A cancer model reports 96% accuracy and the manager is thrilled. Why be cautious, and what would you check?
show how to answer it ▸
If ~96% of patients are healthy, a model that always says 'healthy' scores 96% while catching zero cancers. Accuracy is misleading on imbalanced data. Check Recall (you must not miss cancer), plus Precision, F1, and the confusion matrix; consider oversampling the minority class.
